# SpecDomain

**Type:** domain
**Inferred from:** packages/core/src/types.ts, packages/core/src/commands/scaffold.ts, packages/core/src/utils/spec-id.ts
**Confidence:** high

## Description
The core bounded context for feature specifications — the per-feature containers that hold intent, plans, tasks, and artifact relationships. A spec is the unit of work in Relic.

## Key Entities
- **Spec**: A directory under `.relic/specs/<NNN-slug>/` containing `spec.md`, `plan.md`, `tasks.md`, and `artifacts.json`
- **SpecMeta**: Runtime representation of a spec — its ID, path, and artifact relationships (`owns`, `reads`, `touches_files`)
- **SpecId**: Three-digit zero-padded number followed by a slug (e.g. `001-auth`); generated by `nextSpecId` from the highest existing ID

## Relationships
- Consumes SharedArtifact domain — specs declare ownership and read relationships over shared artifacts
- Referenced by IntersectionDomain — intersection detection compares all loaded specs
- Referenced by ContextDomain — spec context is assembled from spec files + referenced artifacts
- Referenced by SpecResolution domain — `use`, `context`, `fix`, `scaffold` all resolve the active spec via a priority chain
