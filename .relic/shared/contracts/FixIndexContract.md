# FixIndexContract

**Type:** contract
**Owned by:** 005-toon-manifest-format
**Created:** 2026-04-13

## Description
The `fixes/manifest.toon` file — a toon-format index of all fix documents in
`.relic/fixes/`. Written by `relic fix-index`. Gives the LLM a single place to
discover open, pending, and solved fixes without reading each fix document.

Uses the **unified 4-field ManifestEntry schema** shared by all three index spaces.

## File location
`.relic/fixes/manifest.toon`

## Line format (one line per fix document)
```
<name> | <file> | <tags> | <tldr>
```
- `name` — fix title from `# Fix: <Title>` in the fix document (fallback: file stem)
- `file` — filename of the fix document (e.g. `2026-04-13-null-session-crash.md`)
- `tags` — space-separated lowercase keywords; empty string if not yet populated
- `tldr` — one-sentence description; empty string if not yet populated

Empty fields are written as empty strings: `Name | 2026-04-13-crash.md | | `

## Example (after LLM has populated fields)
```
# fixes index
Copilot Codex Per-Command Files | 2026-04-13-copilot-codex-per-command-files.md | copilot codex engines misspecification | Copilot and Codex engines wrote single files instead of per-command files.
```

## Example (on first run — empty fields)
```
# fixes index
# WARNING: tags and tldr fields are empty. Ask your LLM to populate them.
Copilot Codex Per-Command Files | 2026-04-13-copilot-codex-per-command-files.md | |
```

## Written by
- `relic init` — writes an empty file (comment header only: `# fixes index`). No entries.
  Does not call `buildFixIndex` — init only bootstraps structure.
- `buildFixIndex` (internal function in `toon-migrate.ts`) — populates entries from
  existing fix documents. Called by:
  - `relic toon-migrate` — explicit migration / regeneration
  - `relic upgrade` — after every upgrade run

## Read by
- `relic search <keywords> --fix` — scores entries by substring match on name and tldr
- `relic search --deep --fix` — returns all entries unfiltered
- `/relic.fix` — to check for existing fixes before creating a new one
- AI agents doing cross-spec triage

## Rules
- Created empty (comment header only) by `relic init` on a fresh project
- Regenerated by `relic toon-migrate` / `relic upgrade` whenever fix documents change
- Tags and tldr are intentionally left empty on generation — populated by user or LLM
- A warning is emitted to console if any entry has empty fields
