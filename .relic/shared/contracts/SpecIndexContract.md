# SpecIndexContract

**Type:** contract
**Owned by:** 005-toon-manifest-format
**Created:** 2026-04-13

## Description
The `specs/manifest.toon` file — a toon-format index of all spec directories in
`.relic/specs/`. Written by `relic spec-index`. Gives the LLM a single place to
discover available specs without traversing individual directories.

Uses the **unified 4-field ManifestEntry schema** shared by all three index spaces.

## File location
`.relic/specs/manifest.toon`

## Line format (one line per spec)
```
<name> | <file> | <tags> | <tldr>
```
- `name` — spec title from `# Spec: <Title>` in `spec.md` (fallback: folder name)
- `file` — spec folder with trailing slash (e.g. `001-auth/`)
- `tags` — space-separated lowercase keywords; empty string if not yet populated
- `tldr` — one-sentence description; empty string if not yet populated

Empty fields are written as empty strings: `Name | 001-auth/ | | `

## Example (after LLM has populated fields)
```
# specs index
Workflow Test Suite | 001-workflow-test-suite/ | test typescript commands | Test suite for all core TypeScript commands.
Agent Permission Config | 002-agent-permission-config/ | engines permission copilot codex | Per-command files and permission configs for AI engines.
Toon Manifest Format | 005-toon-manifest-format/ | toon format manifest tokens | Replaces JSON manifests with compact pipe-delimited toon format.
```

## Example (on first run — empty fields)
```
# specs index
# WARNING: tags and tldr fields are empty. Ask your LLM to populate them.
Workflow Test Suite | 001-workflow-test-suite/ | |
Agent Permission Config | 002-agent-permission-config/ | |
```

## Written by
- `relic init` — writes an empty file (comment header only: `# specs index`). No entries.
  Does not call `buildSpecIndex` — init only bootstraps structure.
- `buildSpecIndex` (internal function in `toon-migrate.ts`) — populates entries from
  existing spec directories. Called by:
  - `relic toon-migrate` — explicit migration / regeneration
  - `relic upgrade` — after every upgrade run
- `templates/prompts/specify.md` — the LLM appends its own entry after creating a spec

## Read by
- `relic search <keywords> --spec` — scores entries by substring match on name and tldr
- `relic search --deep --spec` — returns all entries unfiltered
- AI agents checking which specs exist before `/relic.plan` or `/relic.fix`

## Rules
- Created empty (comment header only) by `relic init` on a fresh project
- Regenerated by `relic toon-migrate` / `relic upgrade` whenever specs change
- Tags and tldr are intentionally left empty on generation — populated by user or LLM
- The `specify.md` prompt writes the new spec's own entry (with tags and tldr) after creation
- A warning is emitted to console if any entry has empty fields
